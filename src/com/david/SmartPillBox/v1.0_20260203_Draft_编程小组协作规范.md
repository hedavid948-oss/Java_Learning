1. 工具链与环境 (Toolchain)

• 通讯与协同: 飞书 (Lark) - 严禁文件私传，统一 Wiki 与多维表格。

• IDE: VS Code + Codex (GitHub Copilot)。

• 建模: SolidWorks (统一零件库命名)。

• 思维导图: XMind (立项前必须出图)。

2. 命名与代码规范 (Naming Convention)

• 文件夹/项目: `小写-连字符` (例: `smart-pillbox-20260203`)。

• Java 类名: `大驼峰` (例: `PillReminder`)。

• 变量/方法: `小驼峰` (例: `getAlarmTime()`)。

• 常量: `全大写_下划线` (例: `MAX_PILLS`)。

3. Git 提交规范 (Git Flow)

• 每日提交: 每天下班前必须 `push`，未完成写 `wip`。

• Commit 格式:

  • `feat`: 新功能

  • `fix`: 修复

  • `docs`: 文档

  • `refactor`: 重构
